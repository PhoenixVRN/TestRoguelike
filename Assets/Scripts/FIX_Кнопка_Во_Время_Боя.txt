✅ НОВАЯ ЛОГИКА ПОКАЗА КНОПКИ
═══════════════════════════════════════════════════════════════

Кнопка START показывается ТОЛЬКО если ВСЕ 3 условия:

1. heroCount > 0 ✓
   → На поле есть герои игрока (Team 0)

2. isPlacementMode ✓
   → Размещение НЕ заблокировано (бой не идёт)
   → gridManager.IsPlacementLocked() = false

3. isInStartPosition ✓
   → objectToMove в начальном положении (X ≈ 959)
   → Не двигается к X=0

═══════════════════════════════════════════════════════════════
🎬 КАК ЭТО РАБОТАЕТ
═══════════════════════════════════════════════════════════════

СЦЕНАРИЙ 1: Расстановка героев
───────────────────────────────────────────────────────────────
  Герои: 0 → Кнопка СКРЫТА ❌
  Поставили 1 героя → Герои: 1
  ├─ isPlacementMode: ✓ (размещение открыто)
  ├─ isInStartPosition: ✓ (X=959)
  └─ Кнопка ПОКАЗАНА ✅

СЦЕНАРИЙ 2: Нажатие START
───────────────────────────────────────────────────────────────
  START нажат
  ├─ gridManager.LockPlacement() → заблокировано
  ├─ objectToMove → X=0 (двигается)
  └─ isPlacementMode: ❌ → Кнопка СКРЫТА ✅

СЦЕНАРИЙ 3: Бой идёт
───────────────────────────────────────────────────────────────
  Бой идёт
  ├─ isPlacementMode: ❌ (заблокировано)
  ├─ objectToMove X=0
  └─ Кнопка СКРЫТА ✅

СЦЕНАРИЙ 4: Победа и рестарт
───────────────────────────────────────────────────────────────
  Победа → Fade → Рестарт
  ├─ gridManager.UnlockPlacement() → разблокировано ✓
  ├─ objectToMove → X=959 (вернулся) ✓
  ├─ Герои респавнились → Герои: 4 ✓
  └─ Кнопка ПОКАЗАНА ✅
